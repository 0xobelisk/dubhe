# Dubhe API Reference

Comprehensive documentation for interacting with Dubhe Engine through the TypeScript client SDK and smart contract interfaces.

## Client SDK API

The Dubhe Client SDK provides a complete TypeScript interface for building frontend applications.

### Installation

```bash
npm install @0xobelisk/dubhe-client
```

### Core Classes

#### DubheClient

Main client class for interacting with Dubhe contracts.

```typescript
class DubheClient {
  constructor(config: DubheClientConfig)
  
  // Component operations
  getComponent<T>(params: GetComponentParams): Promise<T>
  queryComponents<T>(params: QueryComponentsParams): Promise<T[]>
  
  // Transaction operations  
  tx: SystemTransactions
  signAndExecute(txb: TransactionBlock): Promise<TransactionResult>
  
  // Real-time subscriptions
  subscribe<T>(params: SubscribeParams, callback: (data: T) => void): () => void
}
```

### Configuration

```typescript
interface DubheClientConfig {
  network: 'mainnet' | 'testnet' | 'devnet' | 'localnet'
  packageId: string
  endpoint?: string
  options?: ClientOptions
}

interface ClientOptions {
  enableCache?: boolean
  cacheTimeout?: number
  retryAttempts?: number
  subscriptionTimeout?: number
  gasPrice?: number
}
```

## Smart Contract Interfaces

### Component Structure

All components follow the Move struct pattern:

```move
public struct ComponentName has key, store {
    id: UID,
    field1: Type1,
    field2: Type2,
    // ... additional fields
}
```

### System Functions

System functions are the primary way to modify on-chain state:

```move
public entry fun system_function(
    component: &mut ComponentType,
    param1: Type1,
    param2: Type2
) {
    // Implementation
}
```

## Response Formats

### Component Query Response

```typescript
interface ComponentData<T> {
  entityId: string
  componentName: string
  data: T
  lastUpdated: number
}
```

### Transaction Response

```typescript
interface TransactionResult {
  digest: string
  effects: TransactionEffects
  events: Event[]
  gasUsed: number
}
```

## Error Codes

| Code | Description |
|------|-------------|
| `COMPONENT_NOT_FOUND` | Requested component does not exist |
| `INSUFFICIENT_GAS` | Not enough SUI for transaction fees |
| `INVALID_SIGNATURE` | Transaction signature is invalid |
| `NETWORK_ERROR` | Connection to Sui network failed |

## Rate Limiting

- **Query operations**: 1000 requests per minute
- **Transaction operations**: 100 transactions per minute
- **WebSocket subscriptions**: 50 concurrent connections

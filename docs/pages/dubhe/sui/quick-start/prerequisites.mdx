# Prerequisites

Before you start, make sure your local environment matches the supported baseline.

## Version Baseline

| Component         | Minimum  | Recommended       | Why                                                 |
| ----------------- | -------- | ----------------- | --------------------------------------------------- |
| Node.js           | `22.0.0` | latest `22.x` LTS | Required by Dubhe monorepo and `@0xobelisk/sui-cli` |
| pnpm              | `9.0.0`  | latest `9.x`      | Workspace/package scripts rely on pnpm 9 behavior   |
| Sui CLI           | `1.38+`  | latest stable     | Move build/test/publish compatibility               |
| Docker (optional) | recent   | latest stable     | Recommended for localnet/EVE builder flows          |

## Install Tooling

1. Install Node.js: [nodejs.org](https://nodejs.org/en/download/)
2. Install pnpm:

```bash
npm install --global pnpm
```

3. Install Sui CLI: [official Sui guide](https://docs.sui.io/guides/developer/getting-started/)

4. (Optional) Install Docker for containerized localnet flows.

## Verify Versions

Run:

```bash
node --version   # must be >= 22.0.0
pnpm --version   # must be >= 9.0.0
sui --version    # must be >= 1.38
```

## Configure Sui Client Environment

After installing the Sui CLI, you need to configure the client environments. Run the following commands to add different network environments:

### Localnet

```bash
sui client new-env --alias localnet --rpc http://127.0.0.1:9000
```

### Devnet

```bash
sui client new-env --alias devnet --rpc https://fullnode.devnet.sui.io:443/
```

### Testnet

```bash
sui client new-env --alias testnet --rpc https://fullnode.testnet.sui.io:443/
```

### Mainnet

```bash
sui client new-env --alias mainnet --rpc https://fullnode.mainnet.sui.io:443/
```

These commands will set up your Sui client to interact with different network environments. You can switch between environments using `sui client switch --env <environment-alias>`.

# World Upgrades

EVE world contracts on Sui evolve via new package publishes. When the world package changes, Dubhe-based extension projects must refresh configuration and revalidate scripts.

## What changes on upgrade

- New world package ID may be introduced for new APIs.
- Existing on-chain objects are not automatically retagged to the latest package address.
- Script targets and type filters may need updates.

## ID strategy (practical rule)

- **For new functionality:** use the latest published package ID.
- **For legacy object queries:** use the package ID encoded in object type history.
- **For scripts/dApps:** keep package ID configurable, never hardcode in source.

## Upgrade checklist

1. **Pin target world release**

- Track the exact world-contracts version/revision you depend on.

2. **Refresh deployment artifacts**

- Update:
  - `deployments/<network>/extracted-object-ids.json`
  - `deployments/<network>/Pub*.toml` (if used)
  - `test-resources.json` (if your flow depends on seeded objects)

3. **Update environment**

- Recheck and update:
  - `WORLD_PACKAGE_ID`
  - `BUILDER_PACKAGE_ID`
  - `EXTENSION_CONFIG_ID`
  - `SUI_NETWORK`
  - `SUI_RPC_URL`

4. **Rebuild and republish extension**

```bash
pnpm run build:smart-gate
pnpm run publish:smart-gate:testnet
```

5. **Run flow smoke test**

```bash
pnpm run flow:smart-gate
```

6. **Verify read path**

- Recheck GraphQL/gRPC/type filters if your app or indexer uses explicit type strings.

## Localnet note

On a fresh localnet world deployment, extension publish may require a local dependency override in `move-contracts/smart_gate_extension/Move.toml`:

```toml
[dependencies]
world = { local = "../../../world-contracts/contracts/world" }
```

This avoids depending on unpublished or mismatched remote package metadata during `test-publish`.

## Failure patterns after upgrades

- `Object/Type mismatch`: stale package ID in script target or query filter.
- `Move dependency resolution failed`: outdated `Move.toml` dependency reference.
- `Authorise/configure script fails`: extension/world IDs no longer match current deployment.

Use [Troubleshooting](/dubhe/sui/troubleshooting) for operational checks.

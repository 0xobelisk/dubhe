# Dubhe CLI for Sui

Before getting started with Dubhe CLI, install dependencies in your project:

```bash
pnpm install -D @0xobelisk/sui-cli @0xobelisk/sui-common
```

> `@0xobelisk/sui-common` includes shared config types such as `DubheConfig`.

## Common Usage Pattern

Most project-aware commands read `dubhe.config.ts` from the current working directory.

```bash
# Show all commands
dubhe --help

# Show a specific command's up-to-date options
dubhe <command> --help
```

## Core Commands

### `schemagen`

Generate Dubhe schemas from `dubhe.config.ts`.

```bash
dubhe schemagen [--config-path <path>] [--network <network>]
```

### `build`

Build your Dubhe Move contracts.

```bash
dubhe build --network <network> [--config-path <path>] [--dump-bytecode-as-base64]
```

### `test`

Run Move unit tests for Dubhe contracts.

```bash
dubhe test [--config-path <path>] [--test <test_name>] [--gas-limit <limit>] [--trace]
```

Commonly used advanced options:

- `--profile-gas`: enable gas profiling summary.
- `--profile-source-map-out <path>`: write gas-to-source mapping JSON.
- `--profile-html-out <path>`: write HTML gas report.
- `--profile-flamegraph-out <path>`: write SVG flamegraph report.
- `--profile-baseline <path>`: compare with gas baseline.
- `--fork-fixture <path>`: run fork drift checks from fixture.

### `debug`

Run tests with deep debug output and source hints.

```bash
dubhe debug [--config-path <path>] [--filter <test_name>] [--gas-limit <limit>] [--trace]
```

Common output options:

- `--session-out <path>`: write structured debug session JSON.
- `--session-html-out <path>`: write HTML debug session report.
- `--replay-script-out <path>`: write replay shell script.
- `--show-abort-hints`: print extracted abort hints.
- `--source-hints`: map aborts to local Move source.

### `publish`

Deploy Dubhe contracts to Sui.

```bash
dubhe publish --network <network> [--config-path <path>] [--gas-budget <number>]
```

### `upgrade`

Upgrade deployed Dubhe contracts.

```bash
dubhe upgrade --network <network> [--config-path <path>]
```

## Debug Workbench and Trace Workflow

### `workbench`

Merge debug/gas/trace/fork artifacts into interactive HTML.

```bash
dubhe workbench [--out <file>] [--strict-files <bool>]
```

By default it reads artifacts from `.reports/move/*` and writes:

- `.reports/move/workbench.html`

### `debug-open`

Prepare VSCode Move Trace Debugger launch config and open the best trace target.

```bash
dubhe debug-open [--config-path <path>] [--trace-file <file>] [--source-file <file>]
```

Common options:

- `--launch-path <file>`: launch.json path (default `.vscode/launch.json`).
- `--open <bool>`: whether to open VSCode after config update.
- `--start-debug <bool>`: trigger debug start command in VSCode.
- `--strict <bool>`: fail if no `.json.zst` trace is found automatically.

### `debug-tui`

Open terminal interactive debugger over workbench artifacts.

```bash
dubhe debug-tui [--strict-files <bool>] [--category <cat>] [--severity <level>] [--search <text>]
```

Use it for quick triage without opening a browser/IDE.

### `trace`

Trace transaction digest and optionally replay with dry-run consistency checks.

```bash
dubhe trace --digest <txDigest> [--network <network>] [--replay]
```

Common outputs:

- `--md-out <file>`: markdown trace report.
- `--html-out <file>`: HTML trace report.
- `--call-graph-out <file>` / `--call-graph-json-out <file>`.
- `--consistency-out <file>`: chain-vs-replay consistency report.

### `snapshot`

Capture and diff object snapshots for state auditing.

```bash
dubhe snapshot [--network <network>] [--out <file>] [--diff-with <file>]
```

## Quality and Reliability Commands

### `coverage`

Run coverage checks and threshold enforcement.

```bash
dubhe coverage [--config-path <path>] [--threshold <pct>]
```

### `fuzz`

Run repeated seeded random tests and report flaky seeds.

```bash
dubhe fuzz [--config-path <path>] [--runs <n>] [--seed <seed>]
```

### `invariant`

Run seeded invariant loops with corpus replay and minimization support.

```bash
dubhe invariant [--config-path <path>] [--runs <n>] [--seed <seed>]
```

### `quality-trend`

Aggregate gas/coverage/fuzz metrics into trend timeline and enforce gates.

```bash
dubhe quality-trend [--in <dir>] [--out <file>]
```

## Environment and Utility Commands

### `node`

Manage local Sui node.

```bash
dubhe node [--data-dir <path>] [--force]
```

### `faucet`

Request tokens from faucet.

```bash
dubhe faucet --network <network> [--recipient <address>]
```

### `wait`

Wait for service(s) to become ready.

```bash
dubhe wait [--url <url>] [--localnet] [--timeout <ms>] [--interval <ms>]
```

### `switch-env`

Switch between network environments.

```bash
dubhe switch-env --network <network>
```

### `info`

Show account and network information.

```bash
dubhe info --network <network>
```

### `generate-key`

Generate a new account keypair.

```bash
dubhe generate-key [--force] [--use-next-public]
```

### `check-balance`

Check account balance.

```bash
dubhe check-balance --network <network>
```

### `config-store`

Store Dubhe config for target network.

```bash
dubhe config-store --network <network> [--config-path <path>] [--output-ts-path <path>]
```

### `load-metadata`

Load package metadata from on-chain package.

```bash
dubhe load-metadata --network <network> [--config-path <path>] [--package-id <id>]
```

### `doctor`

Check local development environment.

```bash
dubhe doctor [--install-all]
```

### `convert-json`

Convert `dubhe.config.ts` into JSON.

```bash
dubhe convert-json [--config-path <path>] [--output-path <path>]
```

### `shell`

Open Dubhe interactive shell.

```bash
dubhe shell
```

### `hello`

Sanity command for quick CLI smoke check.

```bash
dubhe hello
```

## Networks

Supported network values used across commands:

- `mainnet`
- `testnet`
- `devnet`
- `localnet`
- `default` (supported by some read-only commands such as `trace`)

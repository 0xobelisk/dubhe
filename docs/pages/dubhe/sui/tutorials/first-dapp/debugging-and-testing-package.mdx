# Debugging and Testing Package

This page describes the current Dubhe debugging flow for Move packages.

## 1. Run tests quickly

```bash
dubhe test --config-path dubhe.config.ts
```

Useful options:

- `--test <name>`: run a single test.
- `--gas-limit <n>`: set gas limit.
- `--trace`: print VM trace output.

## 2. Run deep debug session

```bash
dubhe debug \
  --config-path dubhe.config.ts \
  --gas-limit 500000000 \
  --trace \
  --session-out .reports/move/debug-session.json \
  --session-html-out .reports/move/debug-session.html \
  --replay-script-out .reports/move/debug-replay.sh
```

This generates machine-readable and human-readable artifacts for later triage.

## 3. Generate gas profiling artifacts

```bash
dubhe test \
  --config-path dubhe.config.ts \
  --trace \
  --profile-gas \
  --profile-source-map-out .reports/move/gas-source-map.json \
  --profile-html-out .reports/move/gas-profile.html \
  --profile-flamegraph-out .reports/move/gas-flamegraph.svg
```

## 4. Build interactive workbench

```bash
dubhe workbench --strict-files false --out .reports/move/workbench.html
```

The workbench merges debug/gas/trace/fork artifacts into one timeline view.

## 5. Open VSCode trace debugging

```bash
dubhe debug-open --open true --start-debug false
```

What it does:

- creates or updates `.vscode/launch.json` with `move-debug` config.
- auto-discovers newest `.json.zst` trace file.
- opens VSCode at that trace/source target.

## 6. Use terminal debugger (TUI)

```bash
dubhe debug-tui --strict-files false
```

This is useful when you want a fast terminal-only debug pass.

## 7. End-to-end local example

Dubhe repo includes a full visual debug example:

- `examples/sui/debug-workbench-case`

Inside that example, run:

```bash
pnpm install
pnpm run debug:collect
pnpm run debug:web
# optional smoke validation for web APIs + frontend shell
pnpm run debug:web:smoke
```

Then open:

- `http://localhost:4173`

The web workbench includes:

- timeline filtering/search
- deep-link URL state for current filters/step (`Copy Link`)
- step inspector with play/next/breakpoint controls
- conditional breakpoints (expression + watch vars + hit count)
- playback speed and range-jump navigation
- trace A/B compare (instruction/function-level diff)
- one-click replay and headless `debug-open`
- replay execution hardening (`/api/debug/run` allowlist + busy lock + audit log)
- call stack / variables / state-diff panels
- gas hotspot heatmap + regression ranking
- exception clustering for warn/error triage
- replay command copy
- trace file list and `debug-open` command copy
- bookmarks/annotations + shared sync (`Pull` / `Push`, revision conflict handling)
- timeline chunk rendering with `Load more` for large traces
- command palette (`Ctrl/Cmd+K`) and keyboard shortcuts (`/`, `j/k`, `space`, `b`, `r`)
- report export (`JSON` / `Markdown` / `HTML`)

## Troubleshooting

- No trace file found:
  run `dubhe debug --trace` first, then retry `dubhe debug-open`.
- Missing artifact warnings in workbench/TUI:
  use `--strict-files false` during partial runs.
- Empty web UI:
  run `pnpm run debug:collect` first to refresh `.reports/move` artifacts.

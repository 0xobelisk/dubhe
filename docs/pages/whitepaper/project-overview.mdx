## Introduction

### Our Origin

**Dubhe** originated in 2023 as an internal incubation project of Obelisk Labs, initially known as _Obelisk Engine_. It became independent in 2025.

As of **March 29, 2026**, Dubhe documentation and product packaging are **Sui-first**, with a layered architecture:

- **Engine**: schema-driven Move contract development workflow
- **Channel**: off-chain realtime execution/query path
- **OS**: governance and coordination layer direction

### Mission

> Empower creators and businesses to build verifiable realtime Web3 applications with pragmatic developer tooling.

### Vision

- Make Move-based application development easier to ship and operate.
- Reduce the gap between on-chain correctness and Web2-like product responsiveness.
- Expand toward broader ecosystem coverage while keeping implementation claims explicit.

## Current Product Scope

### Implemented and Available

- Sui-focused contract and client workflow (CLI, codegen, SDK, indexer, GraphQL stack)
- Dubhe Channel HTTP + SSE integration path for nonce, submit, query, and subscription flows
- Production-oriented operational controls (health/readiness/metrics and runtime governance)

### Directional Roadmap

- broader ecosystem compatibility
- deeper enterprise operations support
- stronger governance and verification mechanisms

Roadmap items are directional and may change with engineering priorities.

## Value Proposition

1. **Unified Sui Developer Workflow**
   Schema-driven generation plus SDKs reduce boilerplate between contracts, indexers, and clients.
2. **Realtime Interaction Path**
   Dubhe Channel provides low-latency off-chain read/write and streaming updates on top of on-chain logic.
3. **Operational Readiness**
   Runtime admission limits, SSE governance, and readiness/metrics endpoints improve production control.
4. **Composable Architecture**
   Engine, Channel, and OS are designed to evolve independently while preserving integration boundaries.

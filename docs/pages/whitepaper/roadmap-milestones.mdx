## Engineering Priorities (2026)

### 1) Reliability and Operations

- strengthen readiness policies and production runbooks
- improve rollout safety controls and observability coverage
- tighten runtime governance defaults for submit/publish/subscribe paths

### 2) Developer Experience

- reduce setup and upgrade friction for local-to-testnet workflows
- keep generated code, CLI behavior, and docs in lockstep
- publish clearer reference material for realtime channel integration

### 3) Product Surface Expansion

- stabilize v2 runtime interfaces for richer event/query use cases
- define explicit compatibility boundaries for non-Sui ecosystems
- expand support only after primary Sui path remains stable

## KPI Framework

To avoid reporting noise, milestones are evaluated with auditable engineering metrics:

| Dimension         | Example Signal                                   |
| ----------------- | ------------------------------------------------ |
| Stability         | uptime/error budget, incident count, MTTR        |
| Performance       | p50/p95 request latency under defined workload   |
| Reliability       | retry success rates, dead-letter backlog trends  |
| Developer Success | time-to-first-successful-deploy, docs issue rate |
| Adoption          | active projects using released Dubhe workflows   |

No KPI should be interpreted as guaranteed forward performance unless accompanied by reproducible benchmark or usage methodology.

## Risks and Mitigations

### Risks

- technical debt from rapid scope changes
- divergence between roadmap claims and shipped behavior
- ecosystem expansion ahead of operational maturity

### Mitigations

- Sui-first scope discipline before cross-ecosystem expansion
- implementation-first documentation updates per release cycle
- rollout gates and operational checklists for production changes

## Notes

This roadmap is directional and can change based on engineering validation, ecosystem needs, and security priorities.
